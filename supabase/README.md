# ملفات Supabase المعتمدة

## `schema.sql`

هذا هو ملف القاعدة النظيف للمشروع الجديد. شغّله كاملًا مرة واحدة في Supabase SQL Editor.

يشمل الجداول والدوال والسياسات المعتمدة فقط، بدون ملفات ترقيات قديمة أو تعريفات مكررة.

المفتاح المعتمد للعمليات هو:

`(line_no, operation_no)`

ودالة إعادة بناء المتبرعين المعتمدة هي نسخة واحدة فقط:

`donor_rebuild_chunk(p_limit integer, p_cleanup boolean)`

## `repair_existing_project.sql`

ملف اختياري للمشروع القائم فقط عند ظهور تعارضات قديمة أو رغبتك بتثبيت التصحيحات على قاعدة موجودة.

## تحديث تحليل الحملات التسويقية

- للمشروع الجديد: شغّل `schema.sql` أولًا، ثم `campaign_analysis_v2.sql`.
- للمشروع القائم: شغّل `campaign_analysis_v2.sql` مرة واحدة لإضافة الحملات والتكاليف ومحرك المطابقة والتحليل دون التأثير على العمليات الحالية.
- `campaign_analysis.sql` هو الإصدار السابق الخاص بتحليل الأكواد، وقد أُبقي للتوافق فقط.

### ماذا يضيف `campaign_analysis_v2.sql`؟

- جدول `marketing_campaigns` لتعريف الحملة وقواعد ربطها.
- جدول `marketing_campaign_costs` لبنود التكلفة المؤرخة.
- ربط اختياري بين المستهدف والحملة عبر `campaign_id` مع دعم البيانات السابقة المرتبطة بالاسم.
- دوال المعاينة والحفظ والقائمة والتقرير التفصيلي.
- سياسات RLS وصلاحيات صريحة للمستخدمين المسجلين فقط.
