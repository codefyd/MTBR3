# منصة تحليل المتبرعين (نسخة HTML)

منصة داخلية للإدارة لتحليل المتبرعين اعتمادًا على رقم الجوال كمعرّف أساسي.
مبنية بصفحات **HTML/CSS/JS** خالصة تتصل مباشرة بـ **Supabase** — بدون أي خادم أو بناء (لا Node ولا npm).

التبويبات: **العمليات** · **ملفات المتبرعين** · **المستهدفون في الحملات** · **الإعدادات** + تسجيل دخول محمي.

---

## 1) إعداد قاعدة البيانات (مرة واحدة)
1. افتح مشروعك على Supabase ثم **SQL Editor**.
2. انسخ كامل محتوى ملف `schema.sql` والصقه ثم اضغط **Run**.
   - ينشئ الجداول والدوال (تنظيف الجوال، الرفع، إعادة الحساب) وسياسات الأمان (RLS).

## 2) إنشاء مستخدم الإدارة
- في Supabase: **Authentication → Users → Add user** (بريد + كلمة مرور).
- هذا حساب الدخول للوحة. (يُفضّل تعطيل التسجيل الذاتي من إعدادات Auth ليبقى النظام للإدارة فقط.)

## 3) ضبط بيانات الاتصال
- القيم موجودة جاهزة في `assets/config.js` (رابط المشروع + المفتاح العام).
- إن تغيّر مشروعك، عدّل `SUPABASE_URL` و`SUPABASE_KEY` هناك.
- المفتاح هنا **عام (publishable)** وآمن للنشر؛ الحماية الحقيقية عبر تسجيل الدخول + RLS.

## 4) التشغيل

### محليًا
بسبب سياسات المتصفح، شغّل خادمًا ثابتًا بسيطًا (لا تفتح الملف بنقر مزدوج):
```bash
# أي واحد من التالي داخل مجلد المشروع:
python3 -m http.server 8000
# أو
npx serve .
```
ثم افتح <http://localhost:8000>.

### على GitHub Pages
1. أنشئ مستودعًا على GitHub وارفع كل الملفات (بما فيها مجلد `assets` وملف `.nojekyll`).
2. **Settings → Pages → Source: Deploy from a branch → main / root**.
3. افتح الرابط الذي يظهر — سيحوّلك `index.html` لصفحة الدخول.

> **مهم:** في **Supabase → Authentication → URL Configuration** أضف رابط موقعك (مثل `https://username.github.io/repo`) إلى **Site URL / Redirect URLs**.

## كيف يعمل النظام

**رفع العمليات** (تبويب العمليات): ملف Excel بالأعمدة
`رقم، #العملية، الاسم، الجوال، المشروع، كود الإحالة، القيمة، الكمية، المجموع، التاريخ، الوقت`.
- يُقرأ الملف في المتصفح، ويُرسل لقاعدة البيانات التي تنظّف الجوال وتمنع التكرار عبر `رقم + #العملية` (تحديث القيم عند التكرار).
- بعد الرفع تُعاد إعادة احتساب ملفات المتبرعين تلقائيًا.

**ملفات المتبرعين**: تُبنى لكل جوال (أول/آخر تبرع، الإجمالي، **عدد التبرعات = العمليات الفريدة**، المشاريع، الحالة، الفئة، أيام/فترات التبرع، الاستهداف). فلترة على عمود واحد أو عدة أعمدة معًا + زر إعادة احتساب يدوي.

**المستهدفون في الحملات**: ملف Excel بالأعمدة `الجوال، اسم الحملة، تاريخ الاستهداف` — يحدّث `عدد مرات الاستهداف` و`آخر استهداف` في ملف المتبرع.

**الإعدادات**: مدة الخمول (أيام) وفئات العملاء (الاسم + نطاق عدد التبرعات) قابلة للتعديل.

```

campaigns.html      تبويب المستهدفين
settings.html       الإعدادات
schema.sql          قاعدة بيانات Supabase

```

## التقنيات
Supabase (Auth + Postgres + RLS) · Bootstrap 5 RTL · Font Awesome 6 · خط Amiri · مكتبة `xlsx` (CDN). كل المنطق الثقيل داخل قاعدة البيانات — سريع حتى مع آلاف الصفوف لكل رفع.

## تحديثات هذه النسخة

- **لوحة تحكم جديدة** (`dashboard.html`) هي صفحة البداية: تبرعات الشهر/السنة، أفضل يوم بالأسبوع، أعلى يوم تبرعات، المتبرعون الفريدون، ورسوم شهرية وسنوية. تُحمّل ملخص المتبرعين أولًا ثم تقارير العمليات.
- **الخط الأساسي** أصبح الخط المرفق (`assets/fonts/foda.ttf`)، والأرقام إنجليزية في كل الواجهة.
- **جدول المتبرعين**: المشاريع مطوية (تظهر بالنقر)، نص متوسّط بحدود عمودية متقطعة خفيفة، وترتيب تصاعدي/تنازلي بالنقر على رأس العمود.
- **إعادة ضبط الفلترة** بترتيب منطقي مجمّع.

### مهم: أعد تشغيل `schema.sql`
عُدِّلت دالة `recalculate_donors` وأُضيف عمود `no_time_count`. بعد تحديث الملفات:
1. افتح Supabase → **SQL Editor** وألصق كامل `schema.sql` ثم **Run** (آمن — يعيد إنشاء الدوال).
2. من تبويب «ملفات المتبرعين» اضغط **إعادة احتساب الملفات**.

> **عن عدد التبرعات مقابل تحليل الأيام/الأوقات:** العملية تُحسب تبرعًا واحدًا (`الجوال + #العملية`). العمليات التي لا تحمل تاريخًا/وقتًا صالحًا تُحتسب في «عدد التبرعات» لكنها لا تظهر في تحليل الأيام/الأوقات (يُشار إليها بأيقونة بجانب العدد). لذلك: مجموع الأيام = مجموع الفترات = (التبرعات ذات التاريخ)، و(مجموع الأيام + تبرعات بلا تاريخ) = عدد التبرعات.
